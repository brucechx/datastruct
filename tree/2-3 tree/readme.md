# 2-3 树
## 特性
1. 任一节点只能是 2 度节点或 3 度节点，不存在元素数为 0 的节点。
    * 2度节点：包含 1 个元素的节点将只能有 2 个子节点；
    * 3度节点：包含 2 个元素的节点将只能有 3 个子节点；
2. 所有叶子节点都拥有相同的深度（depth）。
3. 元素始终保持排序顺序。

## 相比二叉查找树，2-3 树的优势
1. 2-3 树可以获得更好的渐进查找时间 O(log2 n)。
2. 2-3 树更容易保持树的平衡。

## 不同的平衡查找树的高度（height）
* AVL 树：高度 ≤ (1.44042..)log2 n
* 红黑树：高度 ≤ 2*log2 (n+1)
* -3 树：高度 ≤ log2 n
* 2-3-4 树：高度 ≤ log2 n
